## 🏢 Backend Architecture Overview

This boilerplate utilizes a **4-Tier Layered MVC Architecture** designed for:

- High security via middleware guards and JWT authentication
- Real-time updates via Socket.io
- Enterprise scalability with modular separation
- Strict **Separation of Concerns** to ensure maintainable and testable code

## 📂 Master Folder Structure

```
src/
├── configs/          # DB, Redis, env connection setups
├── controllers/      # HTTP req/res handlers — zero logic
├── database/         # DB initialization and seed helpers
├── deprecated/       # 🗑 Legacy code — DO NOT IMPORT
├── dtos/             # Zod Data Transfer Objects
├── errors/           # Custom error classes
├── events/           # Internal domain event bus (NEW)
├── jobs/             # Cron jobs and scheduled tasks
├── logs/             # Winston + Morgan audit logs
├── middleware/        # Auth, rate limiters, Helmet, CSP
├── models/           # Mongoose / Prisma schemas
├── queues/           # BullMQ producers & consumers (NEW)
├── cache/            # Redis cache abstraction layer (NEW)
├── repositories/     # CRUD — only layer touching the DB
├── routes/           # URL → Controller mapping
├── scripts/          # One-time seed / migration tools
├── services/         # Business rules and core logic
├── sockets/          # Socket.io event handlers
├── storage/          # Local file uploads (Multer)
├── tests/            # Jest unit and integration tests
├── types/            # Shared TS interfaces & enums (NEW)
├── utils/            # JWT, Bcrypt, date helpers
├── workers/          # Worker threads for CPU tasks
└── index.js          # App entry point
```

## 🏛️ Core Layers

| Folder | Role | Responsibility |
|--------|------|----------------|
| `routes/` | Router | The API map. Connects URLs to specific Controllers. |
| `controllers/` | Handler | The waiters. Handle HTTP req/res with **zero** business logic. |
| `services/` | Logic | The brain. Business rules, calculations, and orchestration. |
| `repositories/` | Data | The librarian. **Only** folder allowed to touch the Database (CRUD). |
| `models/` | Schema | The blueprints. Mongoose/Prisma schemas defining data structure. |

## 🔄 The "Golden" Request Flow

All data **must** travel in this exact order. Breaking the flow breaks the architecture.

```
Client
  ↓
routes/         → URL resolution — unknown paths rejected 404
  ↓
middleware/     → JWT auth, rate limiting, Zod DTO validation, Helmet headers
  ↓
controllers/    → Receive validated data, delegate to service, return response
  ↓
services/       → Business logic, calculations, orchestration
  ↓
repositories/   → CRUD operations — ONLY layer with DB access
  ↓
models/         → Schema enforcement — Mongoose validators / Prisma type safety
  ↓
Response → Client  (JSON payload + optional Socket.io real-time push event)
```

### Flow Rules

1. **Never skip a layer** — controllers must not call repositories directly.
2. **Services own logic** — no `if/else` business logic in controllers or repositories.
3. **Repositories own data** — no raw DB queries in services or controllers.
4. **DTOs validate at entry** — all external input must be validated before reaching a controller.
5. **Errors bubble up** — throw custom error classes; let the global error middleware handle responses.

## 🏗️ Example: User Feature Module

```
src/
├── routes/         user.routes.js
├── controllers/    user.controller.js
├── services/       user.service.js
├── repositories/   user.repository.js
├── models/         user.model.js
├── dtos/           user.dto.js
└── tests/          user.test.js
```

### Route → Controller wiring

```js
// routes/user.routes.js
import { Router } from 'express';
import { authenticate } from '../middleware/auth.middleware.js';
import { validateBody } from '../middleware/validate.middleware.js';
import { CreateUserDto } from '../dtos/user.dto.js';
import { UserController } from '../controllers/user.controller.js';

const router = Router();
const ctrl = new UserController();

router.post('/', validateBody(CreateUserDto), ctrl.create);
router.get('/:id', authenticate, ctrl.findById);

export default router;
```

### Service → Repository pattern

```js
// services/user.service.js
export class UserService {
  constructor(private repo = new UserRepository()) {}

  async createUser(dto) {
    const existing = await this.repo.findByEmail(dto.email);
    if (existing) throw new ConflictError('Email already registered');
    const hashed = await bcrypt.hash(dto.password, 12);
    return this.repo.create({ ...dto, password: hashed });
  }
}
```

## 📜 Architecture Rules (Enforced)

- ✅ Controllers delegate to services — never contain logic
- ✅ Services call repositories — never touch `Model` directly
- ✅ Repositories use models — never raw DB drivers outside this layer
- ✅ DTOs validate all external input at the middleware layer
- ✅ All errors must extend the base `AppError` class
- ✅ Socket events live in `sockets/` — never inline in routes
- ✅ Scheduled tasks live in `jobs/` — never `setInterval` in services
- ❌ No imports from `deprecated/`
- ❌ No DB queries outside `repositories/`
- ❌ No business logic in `controllers/`
